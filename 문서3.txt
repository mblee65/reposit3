------------------------------
ISO/IEC 33020에 따른 프로세스 능력 수준 모델
척도     프로세스 속성		등급
수준 1 : 수행된 프로세스
           -> 프로세스의 목적 달성
           PA 1.1: 프로세스 수행	대부분
           (프로세스의 목적이 달성되는 정도를 나타내는 척도)

수준 2 : 관리된 프로세스
          -> 수행된 프로세스가 관리된 방식(계획되고, 감시되고, 조정되는)으로 이행되고, 
              프로세스의 작업 산출물이 적절하게 수립되고, 통제되고, 유지 관리된다.
           PA 1.1: 프로세스 수행	완전히
           PA 2.1: 수행 관리 		대부분
           (프로세스의 수행이 관리되는 정도를 나타내는 척도)
           PA 2.2: 작업 산출물 관리	대부분
           (프로세스에 의해 만들어 낼 작업 산출물이 적절하게 관리되는 정도를 나타내는 척도)

수준 3 : 정립된 프로세스
          -> 관리된 프로세스는 프로세스 성과를 달성할 수 있는 정의된 프로세스를 사용하여 이행된다.
           PA 1.1: 프로세스 수행	완전히
           PA 2.1: 수행 관리 		완전히
           PA 2.2: 작업 산출물 관리 	완전히
           PA 3.1: 프로세스 정의 	대부분
           (표준 프로세스가 정의된 프로세스의 전개를 지원하기 위해 유지되는 정도를 나타내는 척도)
           PA 3.2: 프로세스 전개	대부분
           (프로세스 성과를 달성하기 위하여 표준 프로세스가 정의된 프로세스로서 전개된 정도를 나타내는 척도)

------------------------------
프로세스 수행 지표 두 가지 유형
- 기본 사례 (BP)
- 작업 산출물 (WP)

. 작업 산출물 특징 (Work Product Characteristic)
. 프로세스 평가 모델 (Process assessment model)

------------------------------
프로세스 능력 지표 유형 (Level을 결정하는 요소)
- 일반 사례(GP, Generic Practice)
- 일반 자원(GR, Generic Resource)

------------------------------
- 프로세스 평가 모델 (PAM, Process Assessment Model)
  . PAM을 통해 총 6가지 또는 5가지 레벨로 Process를 평가
- 프로세스 참조 모델 (PRM, Process Reference Model)

* Process assessment model(PAM)을 가지고 ASPICE에서 무엇을 해야 하는지 What를 설명,
  Process reference model(PRM)을 가지고 필요한 How를 기술
